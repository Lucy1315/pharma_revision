# Streamlit UI 함정

위젯 state 관리 / 업로드 / 다운로드 흐름에서 실제로 부딪힌 버그들.

---

## 1) `st.text_input(key=..., value=...)` 첫 렌더 후 무시

- **상황**: 제품 조회 결과로 허가번호·성분명 입력창을 자동 채우려 함.
- **깨진 것**: 첫 렌더엔 `value=` 가 먹는데, 사용자가 한 글자라도 고치면 이후 `value=` 는 **완전히 무시** 되어 자동 덮어쓰기가 안 됨.
- **원인**: Streamlit 은 동일 `key` 의 위젯 상태를 `st.session_state` 에서 관리. `value=` 는 "초기값" 으로만 쓰고, 이후엔 세션 상태를 신뢰한다.
- **해결**: 위젯을 그리기 **전에** `st.session_state[key] = 값` 으로 직접 주입. 그리고 그 뒤에 `st.text_input(key=key)` 만 호출(`value=` 생략).
- **재발 방지**: "위젯 value 가 안 바뀌네" → 거의 항상 이 이슈. `value=` 쓰지 말고 session_state 주입 패턴을 기본으로.

## 2) 제품 시그니처 바뀔 때만 자동 채움 덮어쓰기

- **상황**: 사용자가 자동 채워진 필드를 수정했는데, 재조회 API 가 한 번 더 돌면서 다시 덮어쓰였다.
- **깨진 것**: 사용자 수정분이 날아감.
- **원인**: "자동 채움" 로직이 매 rerun 마다 무조건 동작.
- **해결**: `(item_seq, item_name, company)` 3-튜플을 **제품 시그니처** 로 세션에 기억. 시그니처가 바뀐 경우에만 자동 채움 재실행. 그 외엔 사용자 수정값 보존.
- **재발 방지**: 자동 채움은 **"언제만 덮어쓸지"** 를 명시적으로 정하고 들어갈 것.

## 3) 다운로드 버튼이 rerun 에 약함

- **상황**: `st.download_button()` 으로 엑셀·Word 내려주는데, 버튼을 누른 후 어떤 다른 위젯이 rerun 을 트리거하면 다운로드가 사라지거나 다시 생성됨.
- **깨진 것**: 사용자 경험 혼란 — 다운로드가 두 번 시작되거나, 클릭 직후 파일이 사라짐.
- **원인**: rerun 때 `download_button` 이 새 buffer 를 갖고 다시 렌더됨. 대용량 엑셀 생성이 매 rerun 마다 반복되는 것도 문제.
- **해결**:
  - 엑셀/Word bytes 를 `st.session_state` 에 캐시.
  - 다운로드는 `<a href="data:application/vnd...;base64,...">` 형식의 **base64 data URI 앵커** 로 제공 → rerun 에 내성 있음.
- **재발 방지**: 생성 비용이 큰 산출물은 session_state 캐시 + data URI 링크 조합을 기본으로.

## 4) nonce 기반 완전 새로고침

- **상황**: "처음부터 다시" 버튼을 만들었는데, `st.session_state.clear()` 만으로는 위젯 key 가 살아있어 일부 상태가 남음.
- **깨진 것**: 초기화했는데 이전 선택이 부분적으로 남음.
- **원인**: 위젯 `key` 가 고정이면 새로운 `key` 의 위젯이 그려지지 않아 내부 state 잔존 가능.
- **해결**: `nonce` 값을 session_state 에 두고, 모든 위젯 `key` 에 `f"upload_{nonce}"` 처럼 포함. "다시" 버튼은 nonce 값을 증가 → 모든 위젯이 새 key 로 재생성 → 완전 초기화.
- **재발 방지**: 완전 초기화 UX 가 필요한 앱에선 **nonce 패턴** 을 처음부터 설계에 넣는다.

## 5) README.txt 자동 감지로 업로드 UX 단순화

- **상황**: 사용자가 품목기준코드·보고기간을 매번 손으로 입력.
- **깨진 것**: 실수 많음 (품목코드 오타, 보고기간 오기입) → 보고서 전체가 틀림.
- **원인**: KIDS 원시자료 ZIP 에 `README.txt` 가 함께 들어있는데, 여기에 품목코드·기간이 이미 있다.
- **해결**: 업로드 시점에 README.txt 를 파싱해서 session_state 에 선반영. 사용자는 확인만 하면 됨.
- **재발 방지**: 사용자에게 "다시 묻는" 값이 있으면, 입력 파일 안에 이미 있는지부터 확인할 것.
