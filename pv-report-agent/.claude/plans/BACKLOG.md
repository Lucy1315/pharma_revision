# Backlog — pv-report-agent

프로젝트 후속 작업 단일 진입점. 글로벌 메모리는 휘발성이라 여기에 적립한다.

## 사용법

- **세션 시작 시**: `P0` → `P1` 순으로 훑어 무엇을 할지 정한다.
- **작업 완료 시**: 해당 항목 앞에 `~~` 취소선 + `✅ (커밋해시)` 추가, 완료일 기록.
- **새 항목 추가 시**: 우선순위(P0/P1/P2/Low) + 한 줄 요약 + `맥락` + `완료 기준`.
- **우선순위 기준**:
  - `P0` — 배포/컴플라이언스 블로커. 다음 세션에 반드시 본다.
  - `P1` — 사용자 체감 이슈. 2~3 세션 내 해결.
  - `P2` — 내부 품질 개선. 여유가 생기면.
  - `Low` — 장기 관찰 항목. 트리거 조건 발생 시에만.
